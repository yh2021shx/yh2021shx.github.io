3 解题报告

Problem A.

情况

赛时 赛后
$\color{yellow}{20}$ $\color{green}{100}$

题目大意

对于所有 $j(1 \leq j \leq m)$，求出 $\sum_i^n\binom{j}{a_i}$。

解法分析

通过 $\sum_i^na_i \leq 10^5$，我们得知不同 $a_i$ 的取值个数 $\leq \sqrt{10^5}$。这个东西以前从来没有接触过。做不出来情有可原。

所以我们可以把相同的 $a_i$ 拿到一起计算，直观时间复杂度为 $\mathcal{O}{(n\sqrt{10^5}\log{n})}$，但可以证明为 $\mathcal{O}(n\sqrt{10^5})$。

需要注意的是阶乘的逆元需要预处理，否则会被卡。

Problem B.

情况

赛时 赛后
$\color{green}{100}$ $\color{green}{100}$

题目大意

给定一个图，每走一步后图的每一条边的权值 $w$ 会变成 $\frac{1+x}{1-x}\bmod p$，问从 $1$ 到 $n$ 的最短路。

解法分析

简单手玩一下这个式子，发现迭代两次后 $w \rightarrow -w \bmod p$。所以发现这个函数周期为 $4$，老套路拆点最短路即可。

Problem C.

情况

赛时 赛后
$\color{red}{0}$ $\color{yellow}{20}$

题目大意

给定一颗树和一个 $k$，会执行 $k$ 次操作，每次操作会把原来的树复制到新树的叶子结点上，然后给定 $q$ 次询问，求出 dfn 为 $x, y$ 的两点的最短路。

解法分析

首先 $k = 0$ 显然暴力 lca。

然后对于 $k \leq 30$，容易想到把原树缩成点，我们就可以在 $\mathcal{O}(1)$ 的时间内计算出一棵树的根的子树大小。若每次要定位一个 dfn 为 $x$ 的子树，可以从 $1$ 开始往原树叶子结点跳，二分跳到哪个叶子结点即可。这样可以 $\mathcal{O}(k\log{n})$ 定位。最后我们可以通过类似倍增 lca 的方法，先让两个点深度相同再一起往上跳，跳到同一颗子树再来一遍普通 lca 即可。复杂度 $\mathcal{O}(k + \log{n})$。

对于 $k \leq 10^9$，容易发现大部分点都是没用的。排除冗余后只有左下角 30 层和根节点到左下角的一条链是有用的。预处理出这些结点然后套 $k \leq 30$ 即可。

Problem D.

情况

赛时 赛后
$\color{yellow}{35}$ $\color{yellow}{35}$

题目大意

给定 $n$ 堆石子，两个人轮流取，每次可以取出开头和结尾的石子堆中若干个式子，谁不能取就输，问先手是否有必胜策略。

解法分析

题解没看懂 QAQ。

把我会的写出来吧。

首先可以定义 $l_{l,r}$ 表示唯一的 $x$ 使得 ${x, a_{l+1}, \cdots, a_r}$ 时先手必败，$r_{l,r}$ 恰好相反。

先证明为何唯一。若存在 $x < y$ 满足先手必败，则当左边石子堆有 $y$ 个时，先手可以取 $y - x$ 个，则后手必败，矛盾。

然后就考虑 $l_{l,r}$ 的递推，$r_{l,r}$ 同理，不叙述。

设 $x = a_r$，则开始分讨：

  • $r_{l,r} = x$

显然 $l_{l-1,r} = 0$。

  • $x < l_{l - 1, r - 1}, x < r_{l,r}$

若 $l_{l-1,r} = x$，则后手可以走对称操作，这样一定会有先手操作完后，首尾有一堆石子被取完。又有 $x < l_{l - 1,r - 1}, x < r_{l, r}$，则后手必胜，先手必输。故 $l_{l - 1, r} = x$

  • $l_{l - 1, r - 1} > r_{l, r}, r_{l,r} < x \leq l_{l - 1, r - 1}$

不会 TAT。

  • $l_{l - 1, r - 1} < r_{l, r}, l_{l - 1,r - 1} \leq x < r_{l, r}$

也不会 QwQ。

  • $x > l_{l - 1, r - 1}, x > r_{l, r}$

没想 =。=

4 总结

  1. 时间复杂度没考虑清楚，导致 D 题挂分
  2. 误判 B 为神仙题，导致死磕 A
  3. 没及时检查，导致 C 挂大分
